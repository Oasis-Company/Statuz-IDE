Design Philosophy

The topological paradigm.

Topology over Syntax

Every IDE treats code as text. Statuz treats code as a topological graph. Functions are nodes. Imports are edges. Modules are clusters. Dependencies are relations. This is not a metaphor — it is a computational model, backed by a Rust graph engine with dual-indexed adjacency lists for O(1) traversal in any direction.

Every node is a computation unit
Functions, classes, modules, files — all nodes in a single unified graph
Every edge is a relationship
depends_on, produces, consumes, validates, informs, contains, delegates_to — 7 built-in relation types, infinitely extensible
Every cluster is a bounded context
Fields represent distinct views of the same ecosystem. Cross-field bridges enable bidirectional traversal
Every change has a blast radius
Reverse BFS impact analysis across all fields simultaneously. Know exactly what breaks.
Additive Engineering

Statuz does not replace VS Code — it adds to it. The full VS Code extension ecosystem, debugger, terminal, and language servers remain untouched. The Statuz Layer sits on top, enriching the editor with topological reasoning, AI-powered code understanding, and strategic decision tracking.

Three Primitive Queries

A complete topological understanding of any software system can be reduced to three questions. Traverse, Impact, Path. Everything else — health reports, centrality analysis, diff detection, subgraph extraction — is a composition of these three primitives.

GraphEngine — In-Memory Topological Runtime

Written in Rust, compiled via napi-rs, the GraphEngine is the computational heart of Statuz. It maintains an adjacency-list-based directed graph with triple-redundant edge storage for O(1) traversal in both directions. Three copies of every edge exist simultaneously: the global edge registry, the source node's outgoing adjacency (indexed by relation type), and the target node's incoming adjacency (also indexed by relation type). Cascade removal ensures consistency without garbage collection.

Storage HashMap<NodeId, Node>
Traversal O(1) adjacency lookup
Edge Indexing Triple-redundant
Runtime Rust + napi-rs
.stz Binary Format

Self-contained, encrypted, content-addressable. Magic bytes "STZ\0" identify the format. Content is msgpack-encoded, optionally zstd-compressed (level 3), and ChaCha20-encrypted with an Argon2id-derived key. The Cluster ID is the first 16 bytes of a blake3 hash. The hash covers content only, so metadata can be modified without invalidating it.

Magic: "STZ\0" (4 bytes) Version: 0x0002 (2 bytes, LE) Flags: 0x0000 (2 bytes, LE) Salt: 16 bytes (Argon2id) Content: msgpack . zstd . ChaCha20 Hash: 32 bytes (blake3)
Cluster & Field Architecture

A Cluster owns all nodes centrally. Each Field owns its own edges — representing different views of the same ecosystem. Cross-field bridges are bidirectional: each bridge creates forward and reverse edges simultaneously. Cross-field traversal uses recursive DFS with visited-set deduplication. Impact analysis propagates across all fields via reverse BFS, following bridge edges to discover affected nodes in remote fields.

Cross-Field Traversal Recursive DFS with visited sets for both fields and nodes. Depth-limited. Returns HashMap<FieldId, (Vec<NodeId>, Vec<Edge>)>.
Cross-Field Impact Reverse BFS across ALL fields. When crossing a bridge, follows the reverse bridge to the other field.

Inline Edit Engine

Dual-mode editing: Fast Apply (search/replace blocks) and Slow Apply (full file rewrite). Search/replace has up to 4 retries with block-by-block streaming. Overlap detection prevents conflicting edits. Right-to-left application preserves line numbers. Snapshot-based undo/redo with full state capture. Streaming diff zones with sweep animation.

Autocomplete Engine

Custom LRU cache with O(1) lookups, abort-on-eviction for LLM requests. 500ms debounce, 60s timeout. Four completion types: fill-middle, redo-suffix, next-line, no-predict. Stack-based parenthesis balancing prevents syntax errors. Subsequence matching handles LLM-generated suffix variations.
